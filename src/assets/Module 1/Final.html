1. Automation mindset & definitions

What Is Automation?

Automation is when software completes tasks for you without you having to repeat the same clicks every day. Instead of a person copying data or sending the same email over and over, a system follows a predefined set of steps (a workflow – a series of actions the computer follows).

In business, automation is commonly used to:

  • Send emails automatically after a form is submitted.
  • Move new leads into a CRM (Customer Relationship Management system – where a company stores customer information).
  • Send invoice or payment reminders on a schedule.
  • Notify staff in Discord when an important event happens.

Your goal as a learner is not to become a professional coder. Your goal is to understand how to connect apps together so that the boring, repetitive parts of work happen automatically.

Simple automation example

1. Customer fills in a website form
2. Automation runs
3. Lead is added to a spreadsheet + Discord alert is sent

One event can trigger several actions at once.

Why Businesses Use Automation

  • Save time – repetitive tasks run automatically.
  • Reduce errors – fewer copy/paste mistakes.
  • Improve customer experience – faster, more consistent replies.
  • Increase revenue – more follow-ups, less leakage.
  • Scale without more staff – systems do the boring work.

When you pitch automations to clients later as a seller or builder, these benefits are the core of your value proposition.

2. Types of automations businesses pay for

Categories of High-Value Automations

There are hundreds of possible automations, but most paid work falls into a few clear categories. Here are some examples you'll see often:

1. Lead Capture & Follow-Up

  • Capture leads from website forms into a spreadsheet.
  • Send an instant welcome email and a Discord or Slack alert.
  • Tag the lead in a CRM (Customer Relationship Management system).

2. Sales Pipeline & Proposals

  • Move deals between stages in a pipeline automatically.
  • Create draft proposals or invoices when a deal hits a stage.
  • Notify the sales team when a new opportunity comes in.

3. Customer Support & Service

  • Turn support forms into tickets inside a help desk tool.
  • Send automatic "we received your message" replies.
  • Escalate urgent issues to a special channel.

4. Reporting & Dashboards

  • Pull key numbers every day into a Google Sheet.
  • Send a daily summary of sales or bookings.
  • Update a simple dashboard the owner checks each morning.

5. Internal Operations

  • Onboarding new staff with checklists and automatic emails.
  • Reminders for renewals, licenses, or maintenance dates.
  • Syncing contact lists between tools (for example CRM → email tool).

3. Core building blocks of a workflow

How an Automation Is Structured

Most automations can be broken down into a few simple parts:

  1. Trigger – what starts the workflow.
  2. Input data – information coming into the system.
  3. Actions – what we want the system to do.
  4. Logic – rules, filters, or branches.
  5. Outputs – where the final result is sent.
Example: New lead from website form

Trigger: new form submission →
Action 1: save lead to Google Sheet →
Action 2: send Discord alert →
Output: lead is stored + team is notified.

In n8n, each step in that chain is represented by a node (a block in the visual workflow that does one job). You will learn how to link nodes together to create your first flows.

Think of an n8n workflow like a simple flowchart: trigger → steps → result.

5. n8n setup checklist

Getting Ready – n8n & Test Workspace

Before building anything, you need a place to create and test your flows. This checklist keeps things simple.

  1. Create an n8n account.
    Go to n8n.io and sign up for a free account. Use an email you can access easily.
  2. Log in and create a new workflow.
    Click Workflows → New. Name it Vault Module 1 – Playground.
  3. Find the Run button.
    In the top-right of n8n you will see a button to Execute Workflow. You will use this a lot while testing.
  4. Turn on "Manual" testing.
    For your first builds we will use manual runs so you can see what is happening step by step.
Your n8n workspace is where all of your Vault workflows will live.
Checkpoint: before moving on, you should be able to log in, create a new workflow, rename it, add a node, and run it manually without errors.
6. Automation #1

Discord Test Notifier

Your first build is a small win: press a button in n8n and get a message in a private Discord channel. This proves everything is connected correctly.

If you're new to Discord: think of a server as a workspace, and channels as rooms inside that workspace. Each channel can have its own settings. A webhook is a special link that lets tools like n8n send messages into a channel.

Step 0 – Create a Discord Webhook (Beginner Friendly)

  1. Open Discord (desktop app or browser) and log in.
  2. On the left, click the server you want to use. If you don't have one yet, click the + icon and choose Create My Own.
  3. Create or choose a text channel where your test messages should go. We recommend something like #vault-module-tests so you don't spam real chats.
  4. Click the channel name → Settings → Integrations → Create Webhook.
  5. Give it a name like Vault Test Bot and choose the correct channel.
  6. Click Copy Webhook URL and paste it into a notes app for now – you'll use it in n8n.
  7. If you want more detail, Discord has an official guide: Intro to Webhooks .
Optional video help: search on YouTube for "Discord webhook for beginners". There are many short, step-by-step videos that show these exact clicks.
Replace this with your own screenshot of the Discord Webhook screen and settings.

Step 1 – Create a new workflow in n8n

  1. Create a new workflow and name it Discord Test Notifier.
  2. Add a Manual Trigger node (this lets you run the workflow by clicking a button).

Step 2 – Add an HTTP Request node

  1. Add a second node and search for HTTP Request.
  2. Set Method to POST.
  3. Paste your Discord Webhook URL into the URL field.
  4. Set Authentication to None.
  5. Set Body Content Type to JSON (a simple text format for data).
  6. In JSON Body, add: 
    {
  "content": "Vault test message – if you can read this, your first automation works."
}
Replace this with a real screenshot of your n8n HTTP Request node pointing to your Discord Webhook.

Step 3 – Connect and test

  1. Connect Manual Trigger → HTTP Request.
  2. Click Execute Workflow.
  3. Check your Discord channel. You should see the test message appear.
Checkpoint: If the message appears in Discord, you have completed Automation #1. If not, re-check your Webhook URL and JSON body.
7. Automation #2

Lead Capture → Google Sheet & Discord

Now you’ll build something closer to a real client automation: when a new lead fills in a form, their details are stored in a spreadsheet and your sales channel in Discord gets a message.

Concept

Flow overview

Trigger: form sends data to a webhook (a special URL that listens for data).
Step 1: clean the data in a Set node.
Step 2: append a new row to Google Sheets.
Step 3: send a formatted message to Discord via HTTP Request.

This is the classic "form → sheet → notification" automation clients pay for.

Step-by-step build

  1. Add a Webhook node.
    • HTTP Method: POST
    • Path: vault-module-lead (for example)
    Copy the Test URL – this is your “form endpoint”.
  2. Add a Set node.
    Map incoming fields into clean keys, e.g.:
    • name
    • email
    • source
    • budget
    • timestamp = {{$now}}
  3. Add a Google Sheets node (Append).
    Select your spreadsheet and worksheet, then map fields from the Set node into each column.
  4. Add a Discord HTTP Request node.
    Same idea as Automation #1, but now send a richer message, e.g.:
    {
  "content": "New lead: {{$json[\"name\"]}} ({{$json[\"email\"]}}) – " +
             "Source: {{$json[\"source\"]}} – Budget: {{$json[\"budget\"]}}"
}
  5. Connect the nodes.
    Webhook → Set → Google Sheets and also a branch from Set → HTTP Request. Both actions should run after Set.
  6. Test with sample data.
    Use a tool like Webhook.site or Postman to send a POST request to your Webhook URL with JSON like:
    {
  "name": "Test Lead",
  "email": "lead@example.com",
  "source": "Module 1",
  "budget": "1-3k"
}
Optional video help: for a deeper understanding of webhooks, check out "Webhooks for Beginners – Full Course" on the freeCodeCamp.org YouTube channel, or any beginner webhook explainer. Credit: all video content belongs to the original creators.
Checkpoint: A new row appears in your Google Sheet and a nicely formatted message appears in Discord whenever you send test data.
8. Automation #3

Daily Accountability Ping

For the final build, you’ll create a simple scheduled automation that sends a daily message. This could be used for personal habits, team stand-ups, or client reminders.

Goal

Every morning at a chosen time, send a Discord message asking for a quick update.

Schedule-based workflow

Cron (time-based trigger) → HTTP Request → Discord channel.
No human needs to remember to send the message – it just happens.

Time-based automations are perfect for reminders, stand-ups, or reporting.

Steps

  1. Create a new workflow called Daily Accountability Ping.
  2. Add a Cron node.
    • Mode: Every Day
    • Time: choose something like 09:00 in your time zone.
  3. Add an HTTP Request node (Discord webhook).
    • Method: POST
    • URL: your Discord Webhook URL
    • Body Content Type: JSON
    • JSON Body: 
    {
  "content": "Daily check-in: what did you get done today?"
}
  4. Connect and activate.
    Connect Cron → HTTP Request, run once manually to test, then turn the workflow Active in n8n.
Checkpoint: The message appears at your chosen time every day without you touching anything. That’s real automation.

10. Key terms

Key Terms & Glossary

Use this page whenever a word feels confusing. You don't need to memorise everything — this is your quick reference.

Automation
Using software to complete tasks automatically without someone having to repeat the same clicks every time.
Workflow
A series of steps that an automation follows, usually shown as a flowchart in tools like n8n.
Node
A single block in an n8n workflow that does one specific job (for example "send HTTP request" or "add row to Google Sheet").
Trigger
The event that starts an automation (for example "new form submitted" or "every day at 09:00").
Action
What the automation does after it is triggered, such as sending a message or saving data.
CRM (Customer Relationship Management)
A system where businesses store and manage information about their customers, leads and interactions (for example HubSpot or Salesforce).
Webhook
A special URL that waits for data. When another system sends data to that URL, it can start an automation instantly.
API (Application Programming Interface)
A set of rules that lets two apps talk to each other and share data. Automation tools use APIs to connect different services.
HTTP (HyperText Transfer Protocol)
The main way data moves across the web. When you send an HTTP request, you are asking another system for something or sending it information.
JSON (JavaScript Object Notation)
A simple text format for sending data, often used in APIs and webhooks. It looks like key–value pairs inside curly braces.
Discord Webhook URL
A special link Discord generates for a channel. When you send a request to this link, a bot message appears in that channel.
Cron
A scheduling system used in n8n to run workflows at specific times, such as every day at 09:00.
Google Sheet (Spreadsheet)
A grid of rows and columns inside Google Sheets where you can store and update data, similar to Excel.
Lead
A person or company who has shown interest in a product or service and may become a customer.
Form endpoint
The URL where form data is sent when someone clicks "Submit". In this module, your Webhook URL acts as the endpoint.
How to use this page: When you see a word in the module that you don't fully get, pause, flip to this glossary, read the definition, then go back. Don't rush – understanding the language is part of becoming valuable as an automation builder.
